Kontextabhängiges Label: „Info ergänzen" statt „Datenfehler melden"

Gleicher Button, gleicher Ort (unten in der Detailansicht), aber Text + Icon hängen davon ab, was tatsächlich fehlt bzw. bestritten wird — keine neue Interaktion, kein Dialog, nur eine andere Beschriftung je nach Datenlage.

Logik: Ist bei Eingang oder Toilette der Wert „Nein" gesetzt → „Datenfehler melden" bleibt (es gibt eine konkrete Aussage, die bestritten werden kann). Ist keiner „Nein", aber mindestens einer „Unbekannt" → „Info ergänzen" (es gibt nichts zu korrigieren, nur etwas beizutragen). Sind beide positiv → Standardtext „Datenfehler melden" bleibt als allgemeine Kontaktmöglichkeit.

Keine Änderung am Klickverhalten selbst — öffnet weiterhin dasselbe Tally-Formular mit denselben Hidden Fields. Nur Label + Icon wechseln. Formular-Vorbefüllung mit sichtbarem Kontext („Aktuell zeigen wir: Toilette = Unbekannt") wäre ein separater, größerer Schritt (Vorschlag #3) und ist hier bewusst nicht enthalten.
