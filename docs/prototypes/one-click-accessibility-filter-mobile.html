Ein-Klick-Filter „Nur bekannt barrierefreie" — Mobile-Prototyp

Ersetzt kein neues Filtersystem — setzt/entfernt nur die bestehenden Filter entrance/toilet/acceptUnknown auf einen Klick. Zwei Zustände: Standard (AN, entspricht dem heutigen App-Default) und AUS (Filter umgangen, wie heute schon bei der Namenssuche).

Zustand AN — Standard

Toggle aktiv, Icon primärfarben. Da Eingang+WC schon gefiltert sind, ist die Trefferliste „sauber" — zusätzlich macht die kleine Zahl rechts sichtbar, dass gerade etwas ausgeblendet wird, statt es unsichtbar zu tun.

Wie das auf bestehenden Code abbildet

  • Toggle AN = heutiger DEFAULT_FILTERS (entrance:true, toilet:true, acceptUnknown:false) — keine neue Filterlogik, nur ein sichtbarer Name dafür.
  • Toggle AUS = Filter serverseitig umgehen, analog zum bestehenden nameHint-Bypass in api/search/route.ts:512-514.
  • „X ausgeblendet" kommt direkt aus dem bereits berechneten filterDebug.total − filterDebug.passed (api/search/route.ts:500-505) — aktuell nur im Empty-State genutzt, hier zusätzlich immer sichtbar, wenn > 0.
  • Parkplatz/Sitzplatz bleiben bewusst außen vor (siehe Diskussion zur Datenlage) — Toggle betrifft nur Eingang + WC.
  • Platzierung: eigene Zeile unter dem bestehenden Ergebnisse-Header (ResultsList.tsx:186-232), oberhalb der Sortierleiste — kein Eingriff in FilterPanel nötig, beide Mechanismen schreiben in denselben SearchFilters-State.
